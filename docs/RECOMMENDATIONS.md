#### 1.1 코드 스플리팅 (30분) - 빠른 효과
**왜 추천하는가?**
- 지도 컴포넌트는 초기 로딩에 필요하지 않음
- 번들 크기 감소로 초기 로딩 속도 개선
- 구현이 간단하고 즉시 효과를 볼 수 있음

**작업 내용:**
- `components/naver-map.tsx` → `dynamic(() => import(...), { ssr: false })`
- `components/tour-detail/detail-map.tsx` → 동일하게 처리
- 사용하는 페이지에서 `Suspense`로 감싸기

**예상 효과:**
- 초기 번들 크기 20-30% 감소
- First Load JS 크기 감소

#### 1.2 API 응답 캐싱 (1-2시간) - 중기 효과
**왜 추천하는가?**
- 같은 데이터를 반복 요청하는 경우가 많음 (필터 변경, 페이지 이동)
- React Query 도입으로 사용자 경험 개선
- 서버 부하 감소

**작업 내용:**
- `@tanstack/react-query` 설치
- QueryClient Provider 설정
- 기존 Server Actions를 React Query hooks로 래핑
- 캐싱 전략 설정 (staleTime: 1시간)

**예상 효과:**
- API 호출 횟수 감소
- 페이지 전환 시 즉시 데이터 표시
- 네트워크 사용량 감소
