## 다음 단계 추천 (PRD 기반)

### 🎯 우선순위 1: Phase 2.3 - 필터 기능 추가

**목표**: 사용자가 지역과 관광 타입으로 관광지 목록을 필터링할 수 있게 함

**작업 내용**:
1. `components/tour-filters.tsx` 생성
   - 지역 필터 UI (시/도 선택)
     - `getAreaCodes()` API로 지역 목록 조회
     - "전체" 옵션 제공
   - 관광 타입 필터 UI (12, 14, 15, 25, 28, 32, 38, 39)
     - `getContentTypeName()` 함수 사용
     - "전체" 옵션 제공
   - 필터 초기화 버튼
   - 정렬 옵션 UI (최신순, 이름순)
   - Sticky 필터 영역 (스크롤 시 상단 고정)
   - design.md 스타일 적용

2. 필터 동작 연결
   - React 상태 관리 (useState)
   - 필터 변경 시 `getAreaBasedList()` API 재호출
   - 필터 파라미터 전달

3. `app/page.tsx` 통합
   - 필터 컴포넌트 추가
   - 필터 상태와 목록 상태 연동

**왜 먼저?**:
- 현재 서울 관광지만 하드코딩되어 있음
- 사용자가 원하는 지역/타입으로 필터링할 수 있어야 함
- MVP 핵심 기능 중 하나 (PRD 2.1)
- 검색 기능보다 먼저 구현하는 것이 자연스러움

**예상 작업 시간**: 2-3시간

### 🎯 우선순위 2: Phase 2.4 - 검색 기능 추가 (MVP 2.3)

**목표**: 키워드로 관광지를 검색할 수 있게 함

**작업 내용**:
1. `components/tour-search.tsx` 생성
   - 검색창 UI (헤더 또는 상단 고정)
   - 엔터 키 또는 검색 버튼 처리
   - design.md 스타일 적용

2. 검색 API 연동
   - `searchKeyword()` Server Action 사용
   - 검색 결과 개수 표시
   - 결과 없음 시 안내 메시지

3. 검색 + 필터 조합
   - 키워드 + 지역 필터
   - 키워드 + 관광 타입 필터

**왜 두 번째?**:
- 필터 기능이 완성된 후 검색 추가가 자연스러움
- MVP 핵심 기능 중 하나 (PRD 2.3)
- 사용자가 원하는 관광지를 빠르게 찾을 수 있게 함

**예상 작업 시간**: 1-2시간

### 🎯 우선순위 3: Phase 2.5 - 지도 연동 (MVP 2.2)

**목표**: 네이버 지도에 관광지 마커 표시

**작업 내용**:
1. `components/naver-map.tsx` 생성
   - 네이버 지도 API 초기화 (테스트 페이지 참고)
   - 관광지 마커 표시
   - 마커 클릭 시 인포윈도우
   - 리스트-지도 연동

2. 반응형 레이아웃
   - 데스크톱: 리스트(좌측 50%) + 지도(우측 50%)
   - 모바일: 탭 형태로 리스트/지도 전환

**왜 세 번째?**:
- 목록과 필터가 완성된 후 지도에 표시하는 것이 자연스러움
- MVP 핵심 기능 중 하나 (PRD 2.2)
- 테스트 페이지(`/map-test`)에서 이미 검증 완료

**예상 작업 시간**: 2-3시간
